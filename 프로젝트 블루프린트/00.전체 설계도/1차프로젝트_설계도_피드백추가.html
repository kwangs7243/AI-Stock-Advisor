AI 뉴스 감성분석 기반 주식 어드바이저 서비스

주식 초보자와 바쁜 사용자가 자극적인 정보에 휘둘리지 않고, 뉴스·감성점수·예측근거·예측 실패 피드백을 함께 확인하며 더 주도적으로 판단할 수 있도록 돕는 서비스 설계 초안

1. 프로젝트 목적

특정 종목과 관련된 뉴스를 수집하고 AI로 감성분석하여, 사용자가 최근 뉴스 흐름과 투자 판단에 참고할 수 있는 정보를 쉽게 확인하도록 돕는다.

핵심 문장:
감성분석은 최종 목적이 아니라, 주가 판단을 돕는 데이터 특성을 만드는 과정이다.

2. 타겟 사용자

주식 초보자 뉴스 확인이 어려운 사용자 리딩방 의존을 줄이고 싶은 사용자 종목 분위기를 빠르게 보고 싶은 사용자

사용자는 종목을 검색하고, 관련 뉴스의 감성 흐름과 AI 설명을 확인하면서 투자 판단에 참고할 수 있다.

5. DB가 필요한 이유

  • 뉴스 원본과 전처리 결과를 저장해야 한다.
  • 감성분석 결과를 뉴스와 연결해야 한다.
  • 주가 데이터와 감성점수를 날짜 기준으로 결합해야 한다.
  • 회귀모델의 예측 결과를 서비스 화면에서 조회해야 한다.
  • LLM이 설명할 때 사용할 근거 데이터를 저장해야 한다.
  • 사용자 검색 기록과 질문 로그를 남길 수 있다.

9. 모델 신뢰성 한계와 피드백 설계

본 프로젝트는 실제 투자 판단을 대신하는 서비스가 아니라, 뉴스 감성분석과 주가 데이터를 결합해 예측 모델을 실험하고 그 결과를 사용자가 이해할 수 있도록 설명하는 학습·실험 목적의 프로젝트이다.

중요 전제:
주식시장은 뉴스 감성만으로 설명하기 어렵고, 기업 실적, 거시경제, 금리, 환율, 수급, 투자 심리, 예측 불가능한 사건 등 다양한 요인에 의해 움직인다. 따라서 본 서비스의 예측 결과는 투자 권유가 아니라 참고용 분석 결과로 제공한다.
구분 내용 프로젝트에서 다루는 방식
예측 한계 뉴스 감성점수와 일부 주가 특성만으로는 실제 주가 등락을 완전히 설명하기 어렵다. 예측값과 함께 신뢰도/주의 문구를 표시하고, 투자 판단을 대신하지 않는다고 명시한다.
오답 분석 모델 예측과 실제 주가 흐름이 다를 수 있다. 예측 실패 사례를 기록하고, 왜 틀렸는지 분석하는 피드백 항목을 추가한다.
추가 데이터 필요성 거래량, 실적, 공시, 금리, 환율, 시장지수, 업종 흐름 등이 부족하면 예측력이 낮아질 수 있다. “어떤 데이터가 추가되면 더 나아질 수 있는가?”를 개선 방향으로 정리한다.
LLM 역할 제한 LLM이 임의로 투자 조언을 생성하면 위험하다. LLM은 DB에 저장된 뉴스, 감성점수, 예측 결과만 근거로 설명하도록 제한한다.
학습 목적 강조 상용 투자 서비스 수준의 정확도를 목표로 하기 어렵다. 데이터 수집 → 모델 학습 → 예측 → 실패 분석 → 개선 방향 도출의 전체 과정을 학습 목표로 삼는다.

10. 예측 결과 피드백 기능 후보

기능 설명 활용 목적
예측 결과 저장 모델이 특정 날짜에 생성한 예측값과 사용한 특성을 DB에 저장한다. 나중에 실제 주가와 비교하기 위한 기준 데이터 확보
실제 결과 비교 예측 시점 이후 실제 주가 등락률과 예측값을 비교한다. 모델이 맞춘 사례와 틀린 사례 구분
오차 분석 예측값과 실제값의 차이를 계산한다. MAE, RMSE, 방향성 정확도 등 평가 지표 확인
실패 원인 메모 예측이 틀린 경우 당시 반영하지 못한 요인을 기록한다. 실적 발표, 공시, 금리, 시장 전체 하락, 돌발 이슈 등 개선 포인트 발견
개선 데이터 제안 현재 모델에 부족한 특성을 정리한다. 추후 고도화 방향 제시 및 발표 자료 근거 확보
LLM 피드백 요약 예측 실패 원인과 개선 방향을 LLM이 쉬운 문장으로 요약한다. 사용자와 발표자가 모델 한계를 이해하기 쉽게 설명
발표에서 강조할 포인트:
이 프로젝트의 가치는 “주가를 정확히 맞히는 것”만이 아니라, 모델이 어떤 데이터로 판단했고, 왜 틀렸으며, 어떤 데이터를 추가하면 더 나아질 수 있는지 분석하는 과정에 있다.
