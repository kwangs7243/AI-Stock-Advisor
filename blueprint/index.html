최종 구현 상태

영역판정확인된 사실
주가·뉴스 수집 및 적재운영 로그 확인Linux 서버의 LIVE_BRIEFING·DAILY_PREDICTION 실행 로그 7건이 최종 SUCCESS로 종료됐습니다.
요약·감성 분석구현·운영 레코드 실패 존재SBERT 추출 요약과 KR-FinBERT 감성 분석 결과가 DB에 적재됩니다. 일부 뉴스는 입력 품질·모델 처리 조건으로 요약 실패가 기록됩니다.
Direction·RecentFlow성능 미검증모델 실행과 결과 저장은 확인했으나 저장된 실험에서 다수 클래스 기준선을 안정적으로 넘지 못했습니다.
Flask 화면·읽기 API구현·검증주요 읽기 요청 8건이 연결 DB 환경에서 모두 HTTP 200을 반환했습니다.
인증·즐겨찾기구현·검증 제한이메일 인증 API와 세션 기반 즐겨찾기 코드가 있습니다. SMTP·쓰기·브라우저 E2E는 이번 검증에서 제외했습니다.
LLM·CI·상용 배포MVP 제외LLM 설명, CI, WSGI, health·metrics는 최종 MVP 범위에 포함하지 않았습니다.

서비스 연결 구조

외부 데이터
  → AI 자동화 파이프라인
  → 공용 MySQL 13개 테이블
  → Flask Repository / API
  → 대시보드·종목 상세·검색·AI 분석 카드

AI 저장소와 Web 저장소는 서로를 HTTP로 직접 호출하지 않습니다. DB 스키마가 두 저장소 사이의 데이터 계약입니다.

발표에서 지켜야 할 해석

말할 수 있는 성과

제한된 기간 안에 실제 DB를 경유하는 수집·분석·조회 vertical slice를 구성하고 테스트와 서버 로그로 동작 근거를 남겼습니다.

구분해야 하는 한계

파이프라인 SUCCESS는 모든 레코드 성공이나 예측 정확도를 뜻하지 않습니다. 모델의 실행 가능성과 예측 품질은 별도 지표입니다.

KDT 1차 팀 프로젝트의 교육용 최종 MVP 기록입니다. 투자 판단이나 상용 운영 준비 완료를 의미하지 않습니다.
