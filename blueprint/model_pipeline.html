ML Pipeline

모델 파이프라인

MVP에서는 게임 카테고리를 대상으로 뉴스 수집, 요약, 감성분석, 특성 생성, 회귀모델 3종 비교, Flask 출력까지 연결되는지 검증합니다.

처리 단계

1. 수집 및 전처리

뉴스 수집네이버 뉴스, Google API, BigKinds 후보 중 구현 가능한 1개 소스부터 시작
텍스트 정제HTML 제거, 중복 제거, 결측치 처리
요약/키워드BART 계열 요약 후보, 키워드 추출, 날짜 정리

2. 관련성 및 감성분석

유사도 계산뉴스와 종목/카테고리 유사도 기준값 적용
카테고리 분류MVP에서는 게임 카테고리부터 시작
모델 후보BERT 파생 모델, 요약본 기반 감성분석
결과긍정, 부정, 중립 라벨과 점수

3. 예측 및 분석

특성 생성기준일 이전 3일 데이터를 문제로 구성
회귀모델 학습기준일 주가 또는 주가 변화값을 정답으로 사용
3종 비교회귀모델 후보 3개를 비교해 성능과 한계 확인
결과 분석예측값과 실제 결과의 차이 해석

모델 및 데이터 후보

항목MVP 기준확장 방향
뉴스 수집네이버 뉴스, Google API, BigKinds 중 구현 가능한 1개 소스부터 시작수집원 다중화와 품질 비교
요약BART 계열 요약 모델 후보를 검토하고, 어려우면 요약 대체 처리 적용요약 품질 평가와 금융 도메인 튜닝
감성분석BERT 파생 모델을 후보로 두고 뉴스 원문보다 요약본을 우선 입력으로 사용금융 문맥에 맞춘 감성분석 모델 고도화
유사도 기준뉴스와 종목/카테고리 유사도 기준값을 정해 관련성이 낮은 뉴스는 제외 가능라벨링 기준 자동화와 카테고리별 기준값 조정
카테고리 확장게임 카테고리부터 시작반도체, AI, 바이오 등 카테고리별 모델 확장

회귀모델 데이터셋 구조

회귀모델은 기준일 이전 3일의 뉴스 감성점수와 주가 관련 특성을 문제로 사용하고, 기준일의 주가 또는 주가 변화값을 정답으로 사용하는 구조로 설계합니다. MVP에서는 3개 회귀모델 후보를 비교해 데이터 흐름과 예측 한계를 확인합니다.

1차 프로젝트 저장 파일 후보

파일설명
sentiment_model.pkl 또는 .h5뉴스 감성분석 모델
regression_model.pkl간단한 주가 예측 참고값 생성 모델
tokenizer텍스트 토큰화에 사용하는 자원
scaler.pkl필요한 경우에만 사용하는 기본 스케일링 도구

향후 연구 방향

고도화된 Feature Engineering, 카테고리별 모델 최적화, 시장지수·수급·환율·금리 등 외부 데이터 결합은 1차 프로젝트 이후 성능 개선을 위해 검토할 확장 연구로 분리합니다.
