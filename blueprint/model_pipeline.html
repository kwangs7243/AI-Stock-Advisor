Model Design

모델 설계

모델 설계는 뉴스 처리, 카테고리별 감성모델, Feature 생성, 예측 목표별 회귀모델을 분리해 이후 실험과 확장이 쉬운 구조로 관리합니다.

1. 뉴스 수집

후보MVP 사용 방식
네이버 뉴스구현 가능한 1차 수집원 후보
Google API수집 안정성 비교 후보
BigKinds정제된 뉴스 데이터 확보 후보

2. 뉴스 전처리

중복 제거동일 기사와 유사 제목을 정리합니다.
날짜 정리뉴스 날짜와 주가 기준일을 맞춥니다.
본문 정제HTML, 광고 문구, 불필요한 문자를 제거합니다.

3. 뉴스 처리

요약

BART 계열 후보요약 모델 적용을 검토
대체 처리모델 적용이 어렵다면 제목+본문 일부로 대체

관련성 판단

키워드 추출기업명, 게임명, 산업 키워드 추출
유사도 계산종목/카테고리와 관련성이 낮은 뉴스 제외 가능

카테고리 분류

게임 우선MVP에서는 게임 카테고리만 구현
확장 가능AI, 반도체, 바이오로 확장

4. 카테고리별 감성모델

감성분석 모델은 하나로 고정하지 않고 카테고리별로 관리합니다. MVP에서는 게임 뉴스 모델만 구현합니다.

flowchart TD
  A["뉴스"] --> B["카테고리 분류"]
  B --> C{"카테고리"}
  C --> D["게임 뉴스 모델"]
  C --> E["AI 뉴스 모델"]
  C --> F["반도체 뉴스 모델"]
  C --> G["바이오 뉴스 모델"]
  D --> H["감성점수 생성"]
  E --> H
  F --> H
  G --> H
감성분석 입력

뉴스 원문 전체가 아니라 요약된 뉴스를 우선 입력으로 사용합니다.

5. Feature 생성

Feature 후보의미
감성점수요약본 기반 감성분석 결과
시가, 종가, 고가, 저가기본 가격 정보
거래량시장 관심과 유동성 정보
뉴스 개수해당 기간 이슈 강도
키워드 정보게임명, 기업명, 이슈 키워드

새로운 Feature는 하나의 Feature Set에 추가하는 방식으로 점진적으로 확장합니다.

6. 회귀모델 구조

Feature Set은 동일하게 사용하고, 예측 목표(y)에 따라 회귀모델을 별도로 관리합니다.

flowchart TD
  A["Feature Set"] --> B["다음날 종가 예측 모델"]
  A --> C["다음날 수익률 예측 모델"]
  A --> D["다음날 거래량 예측 모델"]
MVP 원칙

MVP에서는 예측 목표(y)를 하나만 선택합니다. 다중출력 회귀는 MVP 범위에 포함하지 않습니다.

7. 데이터셋 구조

flowchart LR
  A["최근 3일 데이터
감성점수, 시가, 종가, 고가, 저가,
거래량, 뉴스 개수, 키워드"] --> B["입력 X"]
  B --> C["회귀모델"]
  C --> D["정답 y
다음날 종가 또는 다음날 수익률"]

Window Size는 기본 3일로 시작하지만, 이후 실험에서 5일, 7일 등으로 변경할 수 있게 설계합니다.
