증거 해석의 한계

SUCCESS의 범위

최종 SUCCESS는 작업 오케스트레이션이 완료됐다는 뜻입니다. 레코드별 요약 실패, 데이터 결측, 모델 성능은 별도 확인 대상입니다.

코드 기준선 차이

첨부 로그에는 정리 전 경로명 Jobs/crwaling이 나타납니다. 현재 main은 jobs/crawling으로 정리됐고 테스트는 통과했지만, 정리 후 동일 커밋의 Linux 전체 실행 로그는 이번 증거에 포함되지 않았습니다.

외부 SMTP, 쓰기 요청, 실제 브라우저 E2E, CI·상용 배포 검증도 이번 범위에 포함하지 않았습니다.

KDT 1차 팀 프로젝트의 교육용 최종 MVP 기록입니다. 투자 판단이나 상용 운영 준비 완료를 의미하지 않습니다.
