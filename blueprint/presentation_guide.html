팀장이 그대로 가져가 사용할 수 있는 발표 구성

이 페이지는 기술 문서를 읽지 않아도 프로젝트의 문제, 해결 방법, 핵심 결과와 한계를 발표 순서대로 정리할 수 있도록 만든 안내서입니다.

3주 2일 팀 프로젝트 · 2026-07-16 기준 · 상세 기술 근거는 각 문서에서 확인할 수 있습니다.

발표의 중심 문장

“여러 곳에 흩어진 주가와 뉴스를 자동으로 수집하고, AI 분석 결과를 웹에서 한눈에 확인할 수 있도록 연결했습니다.”

발표 전체가 이 한 문장을 설명하도록 구성하면 됩니다. 기술명은 “어떻게 만들었는가”를 설명할 때만 보조로 사용합니다.

30초 프로젝트 소개

“주식 정보를 확인하려면 주가, 뉴스, 시장 분위기를 여러 곳에서 따로 찾아야 합니다. 저희 팀은 이 불편을 줄이기 위해 주가와 뉴스를 자동으로 모으고, 긴 뉴스에서 핵심 내용과 긍정·부정 흐름을 분석해 하나의 웹 화면에서 보여주는 PILOS를 만들었습니다. 3주 2일 동안 데이터 수집, AI 분석, 데이터베이스 저장, 웹 조회까지 전체 흐름을 구현했습니다.”

슬라이드에 바로 넣을 핵심 문장

문제 정의

“주식 관련 정보가 여러 곳에 흩어져 있어 사용자가 종목의 흐름을 한 번에 파악하기 어렵습니다.”

해결 방법

“주가와 뉴스를 자동으로 수집하고 AI 분석 결과를 하나의 웹 서비스에서 제공합니다.”

기술적 성과

“데이터 수집부터 AI 분석, 데이터베이스 저장, 웹 조회까지 전체 서비스 흐름을 연결했습니다.”

결과의 의미

“높은 투자 수익률을 증명한 프로젝트가 아니라, AI 분석 결과를 실제 서비스 화면까지 전달한 프로젝트입니다.”

시연 진행 문장

  1. 대시보드
    “먼저 여러 종목과 테마의 최근 흐름을 한 화면에서 확인할 수 있습니다.”
  2. 종목 상세
    “관심 종목을 선택하면 가격 변화와 뉴스 분위기를 함께 볼 수 있습니다.”
  3. AI 분석 카드
    “AI가 계산한 방향·흐름 참고값과 긍정·부정 뉴스를 함께 제공합니다.”
  4. 검색
    “회사명이나 종목코드로 원하는 종목을 빠르게 찾을 수 있습니다.”

예상 질문과 답변

질문권장 답변
이 서비스가 종목을 추천하나요?아닙니다. 주가와 뉴스 분석 결과를 이해하기 쉽게 보여주는 교육용 분석 서비스입니다.
예측 정확도가 높은가요?모델 실행과 결과 저장까지 구현했지만 현재 실험에서는 단순 기준선을 안정적으로 넘지 못해 참고값으로만 제공합니다.
실제로 자동 실행됐나요?별도 Linux 서버의 첨부 로그 7건에서 전체 작업이 마지막 단계까지 종료된 것을 확인했습니다.
모든 뉴스 분석이 성공했나요?아닙니다. 전체 자동 작업은 완료됐지만 일부 뉴스는 입력 품질과 처리 조건 때문에 개별 실패가 있었습니다.
AI와 웹은 어떻게 연결되나요?AI가 분석 결과를 공용 데이터베이스에 저장하고, 웹 서비스가 그 결과를 읽어 화면에 보여줍니다.

발표에서 피해야 할 표현

사용하지 않을 표현

  • 주가를 정확히 예측합니다
  • 모든 뉴스 분석에 성공했습니다
  • 상용 서비스 수준으로 완성했습니다
  • AI가 매수 종목을 추천합니다

대신 사용할 표현

  • 분석 참고값을 제공합니다
  • 자동화 흐름의 실행을 확인했습니다
  • 교육용 최종 MVP를 구현했습니다
  • 주가와 뉴스 정보를 한눈에 제공합니다

발표 직전 체크리스트

  • 첫 1분 안에 문제와 서비스를 기술 용어 없이 설명했는가
  • 시연할 화면과 순서를 미리 고정했는가
  • 숫자를 사용할 때 어떤 검증 근거인지 확인했는가
  • 예측 모델의 실행과 성능을 구분해 설명했는가
  • 팀 성과와 개인 역할을 섞어 말하지 않았는가
  • 질문이 깊어질 때 상세 기술 페이지를 근거로 사용할 준비가 됐는가

교육용 프로젝트이며 투자 판단이나 수익을 보장하는 서비스가 아닙니다.
