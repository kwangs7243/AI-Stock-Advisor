API contract

현재 구현 API는 /api/stockTable 하나입니다

문서에 있던 검색·뉴스·감성·예측·LLM endpoint는 후보일 뿐 구현되지 않았습니다. 현재 동작과 다음 계약을 구분합니다.

Blueprint Ver1 · Snapshot: 2026-07-10 · 정적 소스 검토 기준

구현됨

MethodPath상태동작
GET/연동 완료Flask HTML shell
GET/api/stockTable?priceDate=YYYY-MM-DD연동 완료 feature날짜별 전체 주가표. 날짜 누락·유효하지 않음은 최신 적재일로 fallback

성공 응답 shape

{
  "stocks": [{
    "stock_code": "000000", "stock_name": "종목명",
    "price_date": "YYYY-MM-DD", "current_price": 0,
    "open_price": 0, "high_price": 0, "low_price": 0,
    "close_price": 0, "volume": 0, "change_rate": 0.0,
    "current_price_display": "0", "change_rate_display": "0.00%",
    "change_rate_class": ""
  }],
  "priceDate": "YYYY-MM-DD",
  "priceDates": ["YYYY-MM-DD"]
}

현행 camelCase 응답과 향후 naming convention은 계약 고정 단계에서 결정합니다.

권장 구현 순서

  1. /api/stockTable 오류·빈 상태·응답 계약과 route test
  2. 종목 검색 또는 선택 API
  3. 뉴스 + 특정 revision 요약 조회
  4. 기간별 감성 집계
  5. 검증된 prediction 조회
  6. 구조화 결과 기반 LLM 설명

모든 endpoint는 상태, 입력, 성공·오류·빈 응답, 데이터 기준일, 모델 버전을 함께 제공해야 합니다.
