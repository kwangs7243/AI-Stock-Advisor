System Architecture

전체 시스템 구조

데이터 수집, 모델 분석, DB 저장, Flask 웹 서비스, LLM 설명 기능이 연결되는 구조입니다.

구성 개요

flowchart LR
  A["뉴스 수집"] --> B["뉴스 전처리"]
  B --> C["감성분석 모델"]
  C --> D["감성 점수"]
  E["주가 데이터"] --> F["특성 결합"]
  D --> F
  F --> G["회귀모델"]
  G --> H["예측 결과"]
  D --> I[("MySQL DB")]
  E --> I
  H --> I
  I --> J["Flask 서버"]
  J --> K["웹 화면"]
  I --> L["LLM 설명"]
  L --> K

주요 계층

Data Layer

news뉴스 원문과 메타데이터 저장
stock_prices일자별 주가 데이터 저장
sentiment_results뉴스별 감성분석 결과 저장

ML Layer

Sentiment Model뉴스 텍스트를 감성 점수로 변환
Regression Model감성점수와 가격 특성으로 예측 참고값 생성
Model Storage모델, 토크나이저, 스케일러 파일 관리

Service Layer

Flask검색, 조회, API 응답 처리
HTML/CSS/JS분석 결과 시각화
LLM모델 결과를 사용자 친화적으로 설명
