홈 연구 방향 전체 구조 기능 요구사항 DB 설계 모델 파이프라인 LLM 설명 API 설계 일정 계획

System Architecture

전체 시스템 구조

MVP 기준으로 데이터 수집, 감성분석, DB 저장, 간단한 예측, Flask 웹 서비스, LLM 설명 기능이 연결되는 구조입니다.

구성 개요

flowchart LR
  A["뉴스 수집"] --> B["뉴스 전처리"]
  B --> C["감성분석 모델"]
  C --> D["감성 점수"]
  E["주가 데이터"] --> F["특성 결합"]
  D --> F
  F --> G["간단한 회귀모델 또는 모의 예측"]
  G --> H["예측 참고값"]
  D --> I[("MySQL DB")]
  E --> I
  H --> I
  I --> J["Flask 서버"]
  J --> K["웹 화면"]
  I --> L["LLM 설명"]
  L --> K

주요 계층

Data Layer

news뉴스 원문과 메타데이터 저장
stock_prices일자별 주가 데이터 저장
sentiment_results뉴스별 감성분석 결과 저장

ML Layer

Sentiment Model뉴스 텍스트를 감성 점수로 변환
Simple Prediction감성점수와 기본 가격 정보로 예측 참고값 생성
Model Storage모델, 토크나이저, 스케일러 파일 관리

Service Layer

Flask검색, 조회, API 응답 처리
HTML/CSS/JS분석 결과 시각화
LLM모델 결과를 사용자 친화적으로 설명
