Architecture

As-Is와 Target을 분리한 통합 구조

AI와 Web은 서로 HTTP 통신하지 않고 현재 같은 MySQL을 직접 바라봅니다. 실선은 연결된 흐름, 점선 설명은 미구현 연결부입니다.

Blueprint Ver1 · Snapshot: 2026-07-10 · 정적 소스 검토 기준

As-Is — 관찰된 현재 구조

20개 테마 수집기 → CSV
                       ┊ 미구현: normalize / DB upsert
공유 MySQL → stock + stock_price → GET /api/stockTable → 날짜별 정렬 표  [연동 완료]
     └→ news → KoBERT 추출요약 → news_summary → KR-FinBERT → sentiment [오프라인 구현]
                                                                    ┊ 미서빙
Prediction R&D ───────────────────────────────────────────────────┊→ Web 분석 UI
LLM 설명 [계획] ───────────────────────────────────────────────────┘
주가표 연동 완료분석 배치 소스예측 R&D누락 연결부

Target — 모두 계획 상태

공통 수집 엔진 + 테마 설정
→ 표준화·검증 → idempotent ingestion → canonical MySQL + migration
→ versioned summary → versioned sentiment → 검증된 prediction persistence
→ 뉴스·감성·예측 Read API → 실제 분석 UI → 구조화 결과 기반 LLM 설명
↘ Job 상태·재시도·로그·metric·contract test·CI·health check

Target은 목표 구조이며 완료 기능을 뜻하지 않습니다. 모델·데이터·스키마·API 버전을 함께 묶고 각 연결부를 contract test로 확인해야 합니다.

핵심 경계

수집 → DB계획CSV 정규화와 중복 안전 upsert orchestration이 없습니다.
분석 → 예측실험 브랜치특정 요약·감성 revision을 고정한 production Job이 없습니다.
분석 → 사용자계획뉴스·감성·예측 API와 실제 분석 화면이 없습니다.
