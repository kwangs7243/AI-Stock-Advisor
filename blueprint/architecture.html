System Architecture

전체 시스템 구조

1차 프로젝트 기준으로 데이터 수집, 감성분석, DB 저장, 회귀모델 기반 예측, Flask 웹 서비스, LLM 설명 기능이 연결되는 구조입니다.

구성 개요

flowchart LR
  A["뉴스 수집"] --> B["뉴스 전처리"]
  B --> C["감성분석 모델"]
  C --> D["감성 점수"]
  E["주가 데이터"] --> F["특성 결합"]
  D --> F
  F --> G["회귀모델 또는 프로젝트 범위 내 예측"]
  G --> H["예측 참고값"]
  D --> I[("MySQL DB")]
  E --> I
  H --> I
  I --> J["Flask 서버"]
  J --> K["웹 화면"]
  I --> L["LLM 설명"]
  L --> K

주요 계층

Data Layer

news뉴스 원문과 메타데이터 저장
stock_prices일자별 주가 데이터 저장
sentiment_results뉴스별 감성분석 결과 저장

ML Layer

Sentiment Model뉴스 텍스트를 감성 점수로 변환
Regression Model뉴스 감성점수와 주가 데이터를 함께 활용해 예측 참고값 생성
Model Storage모델, 토크나이저, 스케일러 파일 관리

Service Layer

Flask검색, 조회, API 응답 처리
HTML/CSS/JS분석 결과 시각화
LLM모델 결과를 사용자 친화적으로 설명

Design Philosophy

감성분석의 역할

감성분석은 프로젝트의 최종 목적이 아니라, 회귀모델에 사용할 Feature를 생성하는 과정으로 설계했습니다.

회귀모델의 역할

회귀모델은 뉴스 감성점수와 주가 데이터를 함께 활용하여 프로젝트 범위 내 예측을 수행하도록 설계했습니다.

LLM의 역할

LLM은 예측 모델이 아니라 분석 결과와 한계를 사용자에게 이해하기 쉽게 설명하는 역할로 제한했습니다.

DB의 역할

DB는 뉴스, 모델 결과, 사용자 서비스가 동일한 데이터를 공유하도록 연결하는 중앙 허브로 설계했습니다.

본 프로젝트는 실제 투자 서비스를 목표로 하기보다 데이터 기반 의사결정을 지원하고, 모델의 가능성과 한계를 함께 분석하는 방향으로 설계했습니다.
