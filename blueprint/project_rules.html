Project rules

완료 주장은 증거와 검증 수준을 동반합니다

연구·목업·계획을 현재 완료 기능처럼 표현하지 않는 것이 Blueprint Ver1의 핵심 규칙입니다.

Blueprint Ver1 · Snapshot: 2026-07-10 · 정적 소스 검토 기준

사실성과 계약

  1. 모든 완료 상태에는 소스 근거와 검증 수준을 기록한다.
  2. 소스 존재와 재현 가능한 실행을 구분하고, 실행하지 않은 것은 ‘검증 완료’라 쓰지 않는다.
  3. 계획을 현재형으로 쓰지 않으며 상태 배지를 함께 표시한다.
  4. DB 스키마는 하나의 canonical source와 versioned migration으로 관리한다.
  5. schema/API 변경은 AI/Web contract test를 동반한다.
  6. model, data, feature schema, code revision을 함께 기록한다.

보안과 사용자 보호

  1. secret, 내부 host, 계정, 비밀번호를 저장소·문서·예제에 넣지 않는다.
  2. .env.example에는 변수 이름만 제공하고 값은 비운다.
  3. 공개 문서에는 remediation 수준만 쓰고 민감 파일·값을 노출하지 않는다.
  4. “실시간 주가” 대신 “DB 최신 적재일 기준 주가”를 사용한다.
  5. “AI 추천” 대신 “AI 분석 참고값”을 사용하고 투자 비권유를 가까이 표시한다.

연구와 릴리스

  1. 기준선보다 낮은 연구 결과도 숨기지 않는다.
  2. notebook 저장 출력과 재실행 결과를 구분한다.
  3. main 미병합 feature를 production 완료로 표시하지 않는다.
  4. 기존 tag를 이동·덮어쓰기·삭제하지 않는다.
  5. 테스트·호환 manifest 없이 여러 저장소를 하나의 release로 선언하지 않는다.

이 Blueprint는 2026-07-10 검토 snapshot이며 구현 저장소의 런타임 보증서가 아닙니다.
