핵심 규칙

번호규칙
1전체 아키텍처는 유지한다.
2MVP는 최소 기능 검증을 목표로 한다.
3카테고리마다 감성모델을 분리한다.
4예측 목표마다 회귀모델을 분리한다.
5Feature는 하나의 묶음으로 관리하고 점진적으로 확장한다.
6새로운 기능은 기존 구조를 수정하기보다 확장하는 방식으로 구현한다.
7구현보다 구조와 유지보수성을 우선한다.
8모든 설계는 향후 실험과 모델 비교가 가능하도록 작성한다.
