작업 흐름

단계작업확인 기준
1feature/* 브랜치에서 개별 기능 작업담당 기능 단위로 실행 가능
2mvp 브랜치로 Merge전체 파이프라인과 충돌 여부 확인
3MVP 검증수집부터 Flask 출력까지 연결 확인

협업 규칙

  • 한 브랜치는 하나의 기능 목표만 담당합니다.
  • 데이터 수집, 모델, Flask 화면 변경은 서로 영향 범위를 짧게 공유합니다.
  • main은 발표 가능한 상태만 유지합니다.
  • Blueprint 변경은 구현 변경과 함께 업데이트합니다.
