LLM plan

LLM은 예측기가 아니라 구조화 결과 설명 계층입니다

현재 구현 코드·API·의존성이 없는 계획입니다. 신뢰 가능한 뉴스·감성·예측 결과가 먼저 저장되고 제공돼야 합니다.

Blueprint Ver1 · Snapshot: 2026-07-10 · 정적 소스 검토 기준
계획

역할과 경계

허용 역할

  • 데이터 기준일·모델 버전·지표를 쉬운 문장으로 설명
  • 근거 뉴스와 감성·예측 결과의 관계 요약
  • 불확실성과 모델 한계를 명시

금지 역할

  • 수익 보장 또는 매수·매도 지시
  • DB에 없는 사실·뉴스·가격 생성
  • 모델 출력을 확정적 미래 사실로 표현

계획 API

POST /api/llm/explain  [계획]
input: stock_code, as_of_date, selected result ids
server lookup: news_summary + sentiment + prediction + model metadata
output: explanation, evidence ids, limitations, generated_at

클라이언트가 임의 수치를 prompt로 주입하지 않고 서버가 저장된 결과를 ID로 조회합니다.

Guardrail 수용 기준

  1. 근거 ID 없는 수치·사실을 출력하지 않음
  2. 기준일, 출처, 모델 버전, 불확실성 표시
  3. “AI 분석 참고값이며 투자 권유가 아님”을 결과 가까이에 표시
  4. prompt injection과 민감정보 노출 테스트
  5. 예측 부재·실패 시 설명을 생성하지 않고 명확한 empty/error 반환
  6. 대표 입력의 hallucination·faithfulness 평가 통과
