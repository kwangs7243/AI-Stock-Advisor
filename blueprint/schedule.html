Delivery gates

날짜가 아니라 통합 증거로 다음 단계에 진입합니다

각 단계는 entry condition, deliverable, exit criterion을 충족한 뒤 다음 단계로 이동합니다.

Blueprint Ver1 · Snapshot: 2026-07-10 · 정적 소스 검토 기준
  1. Security & baseline freeze

    Entry: feature 작업 보존 · Deliverable: 자격증명 조치 계획, AI/Web 호환 manifest · Exit: 기준 커밋과 소유자 승인

  2. Schema & contract freeze

    Entry: 기준선 확정 · Deliverable: canonical schema, migration, model seed, API naming · Exit: AI/Web contract test 통과

  3. Ingestion vertical slice

    Entry: 표준 레코드 계약 · Deliverable: 게임 fixture 정규화와 idempotent upsert · Exit: 재실행 중복 0, 오류 원인 저장

  4. Summary/sentiment hardening

    Entry: ingestion 데이터 · Deliverable: revision·lineage·retry·길이 계약 · Exit: 작은 fixture pipeline smoke 통과

  5. Prediction validation & persistence

    Entry: 고정 평가셋 · Deliverable: leakage-safe 비교와 저장 Job · Exit: 승인 기준선 통과 또는 연구 보류 결정

  6. Read APIs & real UI

    Entry: 신뢰 가능한 stored results · Deliverable: 뉴스·감성·예측 API와 실제 분석 화면 · Exit: loading/error/empty 포함 E2E 통과

  7. Test / CI / deployment

    Entry: 기능 계약 고정 · Deliverable: 자동 테스트, CI, WSGI, health, logs · Exit: release 후보 smoke와 rollback runbook

  8. LLM explanation

    Entry: 구조화 결과와 한계가 안정적 · Deliverable: 근거 기반 설명 API · Exit: 환각·투자 권유 guardrail 평가 통과

예측이 기준선을 통과하지 못하면 이를 숨기지 않고 R&D로 유지하며, 주가·뉴스·감성 조회 MVP를 독립적으로 완성합니다.
