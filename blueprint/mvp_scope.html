MVP Scope

게임 카테고리 MVP 범위

MVP의 목표는 전체 서비스를 완성하는 것이 아니라, 게임 카테고리 하나를 대상으로 데이터 수집부터 Flask 출력까지 최소 흐름이 연결되는지 검증하는 것입니다.

범위 정의

카테고리 제한

첫 구현 대상은 게임 카테고리로 제한합니다. 반도체, AI, 바이오 등은 이후 확장 대상으로 둡니다.

완료 기준

UI 완성도보다 데이터가 수집, 분석, 결합, 모델 학습, Flask 화면 출력까지 전달되는지를 우선 확인합니다.

MVP 처리 흐름

1게임 뉴스 수집
2전처리 및 요약
3감성분석
4특성 생성
5회귀모델 3종 비교
6Flask 결과 출력

구현 범위

영역MVP 기준
뉴스 수집원네이버 뉴스, Google API, BigKinds를 후보로 기록하되, 구현 가능한 1개 소스부터 시작합니다.
감성분석 입력뉴스 원문 전체보다 요약본을 우선 사용합니다. 요약 모델 적용이 어렵다면 요약 대체 처리로 시작합니다.
회귀모델 데이터셋기준일 이전 3일 데이터를 문제로 사용하고, 기준일의 주가 또는 주가 변화값을 정답으로 사용합니다.
회귀모델 비교3개 회귀모델 후보를 비교해 예측 성능과 한계를 확인합니다.
Flask 출력화면 완성도보다 수집 데이터, 감성점수, 예측 결과가 끝까지 전달되는지 확인합니다.

MVP 완료 기준

뉴스 수집게임 뉴스 데이터 수집 가능
전처리뉴스 전처리 가능
요약요약 또는 요약 대체 처리 가능
감성점수요약본 기반 감성점수 생성 가능
데이터 결합주가 데이터와 결합 가능
학습 데이터셋회귀모델용 데이터셋 생성 가능
모델 비교3개 회귀모델 비교 가능
웹 확인Flask 화면에서 결과 확인 가능
