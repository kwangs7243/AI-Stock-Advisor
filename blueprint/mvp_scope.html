MVP Plan

게임 카테고리 MVP 구현 계획

MVP는 게임 카테고리 하나를 대상으로 데이터 수집, 전처리, 감성분석, Feature 생성, 회귀모델 학습, Flask 출력까지 최소 흐름이 연결되는지 검증합니다.

MVP 목표

왜 MVP를 만드는가

전체 서비스를 한 번에 완성하기보다, 가장 작은 범위에서 데이터와 모델, 화면 출력이 연결되는지 먼저 확인합니다.

구현 범위

영역구현 내용
카테고리게임 카테고리로 제한
뉴스 수집네이버 뉴스, Google API, BigKinds 후보 중 구현 가능한 1개 소스부터 시작
뉴스 처리전처리, 요약 또는 요약 대체 처리, 키워드 추출, 유사도 계산, 카테고리 분류
감성분석뉴스 원문보다 요약본을 우선 입력으로 사용
회귀모델최근 3일 데이터를 X로 사용하고, 하나의 예측 목표 y를 선택해 3개 회귀모델 비교
FlaskUI 완성도보다 데이터가 끝까지 전달되는지 확인

구현하지 않는 기능

제외 항목이유
모든 카테고리 동시 구현게임 카테고리에서 먼저 파이프라인을 검증합니다.
다중출력 회귀MVP에서는 하나의 예측 목표만 선택합니다.
고도화된 RAG와 벡터DBLLM 설명은 분석 결과 요약에 집중합니다.
관리자 페이지와 자동 재학습서비스 운영 기능은 파이프라인 검증 이후로 분리합니다.

단계별 작업

단계목적결과
1게임 뉴스 데이터 확보수집 가능한 뉴스 원천 1개 확보
2뉴스 분석 입력 생성전처리, 요약, 키워드, 유사도, 카테고리 정보 생성
3감성점수 생성요약본 기반 감성분석 결과 저장
4회귀모델 데이터셋 생성최근 3일 X와 다음날 y 구조 생성
5회귀모델 3종 비교모델별 결과와 한계 확인
6Flask 출력뉴스, 감성점수, 예측 결과 화면 확인
