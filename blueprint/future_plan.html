Future plan

확장보다 통합과 재현성을 먼저 해결합니다

20개 테마는 이미 수집 코드 자산으로 존재합니다. 앞으로의 핵심은 테마 수를 늘리는 일이 아니라 공통 엔진과 신뢰 가능한 E2E를 만드는 일입니다.

Blueprint Ver1 · Snapshot: 2026-07-10 · 정적 소스 검토 기준
  1. 통합과 재현성

    기준 커밋, 환경변수 이름, 데이터·모델 manifest, 실행 runbook을 고정합니다.

  2. 공통 수집 엔진화

    테마별 복제 스크립트를 공통 엔진 + 테마 설정으로 수렴합니다.

  3. 데이터 품질과 정책

    중복·결측·날짜·URL 계약과 보존기간, 뉴스 저작권, robots/이용약관을 검토합니다.

  4. 예측 검증

    기준선·시간 검증·ablation을 통과한 목표만 서비스 후보로 승격합니다.

  5. 실제 API/UI

    뉴스·감성·예측을 기준일·버전·empty/error 상태와 함께 제공합니다.

  6. 카테고리별 모델

    충분한 데이터와 비교 실험이 있을 때 단일 모델 대비 이득을 검증합니다.

  7. LLM/RAG

    구조화 결과가 신뢰 가능해진 뒤 근거 제한 설명 계층을 추가합니다.

  8. 운영 자동화

    scheduler, 재학습, drift, health, metric, rollback을 단계적으로 도입합니다.

제품 신뢰성 과제

UITOP3·분석 패널을 실제 API 또는 명확한 empty state로 전환
운영production WSGI, health, 구조화 로그, metric, retry
사용자 보호데이터 기준일·모델 버전·불확실성·비권유 표시
