Database Design

DB 설계 초안

1차 프로젝트에서는 뉴스, 감성점수, 주가, 예측 결과를 저장하고, 일부 테이블은 향후 확장을 고려해 설계합니다.

DB가 필요한 이유

분석 데이터 저장

뉴스 원본, 전처리 결과, 감성분석 결과를 일관된 구조로 저장해야 합니다.

서비스 조회 성능

Flask 화면에서 종목별 뉴스, 감성 흐름, 간단한 예측 참고값을 조회해야 합니다.

주요 테이블 후보

테이블 역할 주요 컬럼 후보
stocks종목 기본 정보stock_id, stock_code, company_name, market
news뉴스 원문과 메타데이터news_id, stock_id, title, content, url, publisher, published_at
sentiment_results뉴스별 감성분석 결과result_id, news_id, label, score, model_version
stock_prices일자별 주가 데이터price_id, stock_id, date, close_price, change_rate, volume
predictions1차 프로젝트 예측 참고값 저장prediction_id, stock_id, base_date, predicted_value, model_version
prediction_feedback향후 확장을 고려한 예측 결과 비교 기록feedback_id, prediction_id, actual_value, error_value, note
chat_logsLLM 설명 또는 질문 기록log_id, stock_id, question, answer, created_at

prediction_feedback 테이블은 현재 프로젝트의 자동 피드백 API 구현 대상이 아니라, 예측값과 실제 결과를 비교해 모델의 한계와 개선 방향을 분석하기 위한 향후 확장 설계입니다.
