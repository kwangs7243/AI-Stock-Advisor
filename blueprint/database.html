Blueprint Ver1 · Snapshot: 2026-07-10 · 정적 소스 검토 기준

P0: canonical schema 필요

AI와 Web이 같은 MySQL을 직접 사용하므로 컬럼명·타입·유일키 drift가 곧 통합 장애가 됩니다. DROP snapshot과 versioned migration을 분리하고 contract test를 도입해야 합니다.

Observed draft — 9개 테이블

테이블관찰된 역할연동 수준
theme테마 기준정보소스/스키마
stock종목·테마 기준정보주가표 사용
stock_price일자별 OHLCV·등락률주가표 사용
news제목·원문 URL·정제 본문분석 입력
news_summary모델별 추출요약 결과요약 배치
sentiment확률·파생점수·라벨감성 배치
model모델 메타데이터seed 필요
prediction예측 결과 후보미연동
user회원 후보미연동

확정해야 할 계약

  • stock_code는 선행 0을 보존하는 6자리 문자열
  • URL 컬럼은 original_url로 통일하고 url_hash 유일성 정의
  • model_name + model_version + immutable_revision 유일성과 seed 제공
  • 요약 1,000자 코드와 varchar(500) 불일치 해결
  • 요약·감성 처리에 queued/running/succeeded/failed, attempt, last_error 저장
  • 감성 결과가 사용한 news_summary_id 또는 revision 보존
  • news legacy summary 컬럼과 news_summary 역할 중복 결정

핵심 표준 입력 후보

stock_code, theme_name, company_name, published_at, title
original_url, url_hash, clean_article_text
price_date, open_price, high_price, low_price, close_price
volume, change_rate, source, ingested_at
