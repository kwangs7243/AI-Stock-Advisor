연구 방향

연구형 프로젝트 방향성

본 프로젝트는 주가를 정확히 맞히는 상용 투자 서비스가 아니라, 1차 프로젝트 구현 결과를 통해 뉴스 감성분석 기반 예측의 가능성과 한계를 확인하고 향후 연구 방향을 제시하는 프로젝트입니다.

핵심 연구 질문

뉴스 기반 주가 예측은 어디까지 가능한가?
그리고 어떤 특성이 모델의 예측 성능을 향상시키는가?

단순히 뉴스의 긍정/부정을 분류하는 것에서 끝나지 않고, 현재 프로젝트에서 뉴스 감성점수와 기본 주가 정보를 결합한 예측 기능을 구현합니다. 모든 연구를 구현하기보다, 구현 결과를 기반으로 확인한 한계와 향후 연구 과제를 정리합니다.

프로젝트 관점 전환

기존 관점 연구형 관점
AI로 주가를 맞히는 서비스뉴스와 특성이 예측에 얼마나 기여할 수 있는지 일부 검증하는 프로젝트
예측 정확도만 강조예측 실패 원인과 개선 방향을 결과 분석으로 제시
사용자에게 투자 판단 제공사용자가 데이터와 근거를 확인하도록 돕는 참고 서비스
모델 결과를 정답처럼 제공모델 결과의 한계와 불확실성을 함께 제공

중요한 전제

뉴스 감성만으로 시장을 설명할 수는 없습니다.

주식시장은 실적, 공시, 금리, 환율, 외국인·기관 수급, 시장지수, 업종 흐름, 돌발 이슈 등 다양한 요인이 동시에 영향을 줍니다.

따라서 본 프로젝트의 예측 결과는 투자 권유가 아니라, 모델이 어떤 데이터를 근거로 판단했는지 확인하기 위한 참고용 분석 결과로 제공합니다.

현재 프로젝트에서 구현한 내용

1뉴스 수집
2감성분석
3특성 생성
4회귀모델 학습
5실제 결과 비교
6실패 원인 분석

향후 특성 세트 실험 방향

아래 단계별 특성 비교는 현재 1차 프로젝트에서 모두 구현하는 범위가 아니라, 기본 감성점수 기반 예측 이후 성능 개선을 위해 확장할 수 있는 연구 방향입니다.

단계 사용 특성 예상 목적
Level 1긍정 뉴스 비율, 부정 뉴스 비율, 평균 감성점수뉴스 감성만으로 예측 가능한지 확인하는 기준선
Level 2최근 3일/7일 감성 평균, 감성 변화량, 뉴스 개수감성의 흐름과 변화가 예측에 도움이 되는지 확인
Level 3전일 등락률, 거래량, 이동평균, 변동성기본 주가 특성을 추가했을 때 성능 변화 확인
Level 4시장지수, 업종지수, 외국인/기관 수급, 환율, 금리 등외부 시장 요인을 추가했을 때 신뢰성이 높아지는지 확인

구현 결과로 확인할 수 있는 한계

평가 지표 후보

  • MAE: 예측값과 실제값의 평균 절대 오차
  • RMSE: 큰 오차에 더 민감한 회귀 평가 지표
  • R² Score: 모델이 변동성을 얼마나 설명하는지 확인
  • 방향성 정확도: 상승/하락 방향을 맞췄는지 확인
  • Baseline 비교: 1차 프로젝트 이후 확장 연구에서 단순 기준 모델과 비교

결과 분석 항목

  • 뉴스 감성은 긍정적이었지만 실제 주가는 하락했는가?
  • 시장 전체 하락이나 업종 악재가 있었는가?
  • 실적 발표, 공시, 금리, 환율 같은 정보가 누락되었는가?
  • 뉴스가 이미 주가에 선반영되었는가?
  • 감성분석 모델이 금융 문맥을 잘못 해석했는가?

Prediction Feedback의 의미

Prediction Feedback은 자동 피드백 시스템이나 Ajax 기반 저장 기능이 아니라, 예측 결과와 실제 결과를 비교하여 모델의 한계를 분석하고 향후 개선 방향을 도출하기 위한 연구 개념입니다. 현재 프로젝트에서는 서비스 기능보다 발표 결과 분석의 일부로 다룹니다.

향후 연구로 확장할 수 있는 방향

실험 결과 예측력이 낮게 나오더라도, 어떤 데이터가 부족했는지 분석하면 향후 고도화 방향을 명확하게 제시할 수 있습니다. 이 과정 자체가 프로젝트의 중요한 학습 성과가 됩니다.

기업 실적 공시 데이터 금리 환율 외국인 수급 기관 수급 시장지수 업종지수 SNS 감성 검색량 거래대금 변동성 지표

발표에서 강조할 메시지

본 프로젝트의 목표는 주가를 완벽히 예측하는 것이 아니라, 1차 프로젝트 구현 범위로 뉴스 감성분석 기반 예측 흐름을 구성하고, 그 결과의 한계와 향후 개선 방향을 설명하는 것입니다.

결론

이 프로젝트는 서비스 구현과 모델 학습을 함께 수행하지만, 최종적으로는 1차 프로젝트 구현 범위와 향후 확장 범위를 명확히 구분하는 연구형 프로젝트입니다. 예측 정확도가 높지 않더라도 데이터 수집, 감성분석, 간단한 예측, 결과 분석, 개선 방향 제시의 흐름을 보여줄 수 있다면 1차 프로젝트 완성본으로 충분한 완성도를 가질 수 있습니다.
